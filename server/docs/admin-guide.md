# Admin Guide

All administration is done via CLI commands on the server binary.

## User management

### Add a user

```bash
./shmirat-server add-user alice@example.com
```

Output:
```
User added: alice@example.com
Token: 4fd1ff7a95a9a20e534c3f070021307cde742985e3220c95101033270b2920cf
Note: user is NOT approved yet. Run: shmirat-server approve alice@example.com
```

The token is printed once. Copy it and send it to the user securely. It cannot be retrieved later (only regenerated by creating a new user).

New users are **not approved by default**. Their token will be rejected until approved.

### Approve a user

```bash
./shmirat-server approve alice@example.com
```

The user's token immediately starts working.

### Revoke a user

```bash
./shmirat-server revoke alice@example.com
```

The user's token immediately stops working. Their data (contributions) is retained.

### List users

```bash
./shmirat-server list-users
```

Output:
```
EMAIL                          APPROVED   CONTRIBUTIONS   CREATED
--------------------------------------------------------------------------------
alice@example.com              YES        142             2026-03-09 14:56:57
bob@example.com                NO         0               2026-03-09 15:30:00

Total: 2 users
```

The stats command shows:
- **Vote Distribution**: How many images have single vs consensus votes

- **Vote Sources**: Breakdown by classification origin (local ML, Haiku AI, manual user input) — sourced from the individual votes audit log
- **Cache Efficiency**: How many redundant API/ML calls the shared cache has prevented

## Database

The server stores all data in `server.db` (SQLite) in the working directory. The database is created automatically on first run.

To back up: copy `server.db` while the server is running (SQLite WAL mode makes this safe).

To reset: stop the server and delete `server.db`.
